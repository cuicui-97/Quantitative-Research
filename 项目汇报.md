### 2.2 关键技术创新：PIT数据处理

**什么是PIT（Point-In-Time）？**

PIT要求我们在任何历史时点T，只能使用**当时已知**的数据，不能用未来信息。

**我们的方案：**

```python
# 传统做法（错误）- 用报告期end_date
report_date = "2024-03-31"  # Q1财报截止日
# 问题：财报实际4月25日才公告，3月31日时市场并不知道这份数据

# 我们的做法（正确）- 用公告日ann_date
announce_date = "2024-04-25"  # 实际公告日
# 正确：从4月25日开始使用这份数据，直到下一份财报公告
```

**实现机制：**
- 利润表数据保留 `ann_date`（公告日）字段
- 矩阵构建时按 `ann_date` 展开数据
- 从公告日到下一公告日前，因子值保持不变
- 确保回测时不使用未来数据

### 2.3 数据质量保障

### 3.2 矩阵化计算：高效因子构建的核心

**为什么要用矩阵？**

传统做法：逐日循环，每次处理一只股票 → 慢，代码复杂

```python
# 传统做法（低效）
for date in dates:           # 3000+交易日
    for stock in stocks:     # 5000+股票
        factor = calculate(stock, date)  # 1500万+次循环
```

我们的做法：矩阵化向量化计算 → 快，代码简洁

```python
# 矩阵化做法（高效）
close_matrix  # shape: (3000, 5000) = 日期 × 股票
momentum_20d = close_matrix.pct_change(20)  # 一次性计算，向量化
```

**效率对比：**

| 操作 | 传统循环 | 矩阵化 | 加速比 |
|------|---------|--------|--------|
| 20日动量计算 | ~30分钟 | ~2秒 | **900x** |
| 全市场因子构建 | 数小时 | ~5分钟 | **100x** |
| 回测计算 | 小时级 | 分钟级 | **60x** |

### 3.3 因子构建基类：代码复用的典范

我们提取了**FactorMatrixBuilder基类**，新因子只需20行代码：

```python
# 以动量因子为例
class MomentumBuilder(FactorMatrixBuilder):
    def get_factor_definitions(self):
        return [
            ('reversal_5d', '5日反转'),
            ('momentum_20d', '20日动量'),
            ('momentum_60d', '60日动量'),
        ]

    def calculate_factors_for_stock(self, ts_code, close, **kwargs):
        return {
            'reversal_5d': -close.pct_change(5),
            'momentum_20d': close.pct_change(20),
            'momentum_60d': close.pct_change(60),
        }

# 运行构建
MomentumBuilder().build_and_save()
```

**基类提供的功能：**
- 自动加载原始数据
- 初始化NaN矩阵
- 逐股票并行处理
- 自动保存和统计
- 进度日志输出

## 四、因子分析体系

### 4.1 完整分析流程

```
┌─────────────────────────────────────────────────────────┐
│  单因子分析流程                                          │
├─────────────────────────────────────────────────────────┤
│  1. 数据对齐                                              │
│     - 因子矩阵 + 收益率矩阵 + 可交易矩阵 + 市值矩阵        │
│                                                         │
│  2. 因子分组                                              │
│     - 每日按因子值排序，分为10组                          │
│     - 先过滤不可交易股票，再分组                          │
│                                                         │
│  3. 收益计算                                              │
│     - 等权收益：每组内股票平均收益                        │
│     - 市值加权：按流通市值加权平均                        │
│     - 支持交易成本（佣金+印花税+滑点）                    │
│                                                         │
│  4. 统计分析                                              │
│     - 年化收益率、夏普比率、最大回撤                      │
│     - IC/IR分析（因子与收益的相关系数）                   │
│     - 胜率、盈亏比                                        │
│                                                         │
│  5. 可视化输出                                            │
│     - 累计收益曲线                                        │
│     - 分组收益柱状图                                      │
│     - HTML交互式报告                                      │
└─────────────────────────────────────────────────────────┘
```
